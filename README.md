# 오늘의 크립토 · Today in Crypto

> 디지털 자산, 매일 아침, 한 사람의 시선으로.
> A one-person daily crypto publication — published in both Korean and English.

## 무엇인가

매일 아침 크립토 뉴스를 한 사람의 시선으로 정리해 전하는 **1인 미디어 잡지**.
지금은 다른 것은 일절 하지 않고, **크립토 뉴스를 큐레이션해 전달하는 것 한 가지**에만 집중한다.

핵심 원칙: **뉴스 자체는 어디서나 구할 수 있는 commodity다. 사람들이 구독하는 것은 편집자의 시선이다.**
단순 요약 매체는 이미 수천 개 있고 서로 구분되지 않는다. 이 매체가 살아남는 길은 큐레이션과 목소리에 있다.

> 추상적으로 이 사이트는 **캔버스**다. 지금은 크립토 뉴스만 담지만, 나중에 디파이 실험 같은
> "예상 못 한 그림"도 얹을 수 있도록 — 무엇을 올려도 받아주는 바탕으로 설계한다.

## 벤치마크

### NextDraft — 구조의 뼈대
[NextDraft](https://nextdraft.com/)에서 가져온 것:
- 한 호에 **여러 꼭지**, 각 꼭지마다 **말맛 살린 제목**
- 본문은 직접 취재가 아니라 **좋은 출처에서 핵심을 골라 엮음** (curation, not reporting)
- `+` 패턴: 메인 링크 + 관련 곁가지 링크
- 전체 질감: "한 사람이 인터넷 전체를 대신 읽어준다"

> 단, NextDraft의 "사실 인용 + 괄호 속 사견" 구분은 그대로 쓰지 않는다.
> 우리는 직접 분석(디파이 해부)도 많이 하므로, **전체를 한 목소리로 흐르게** 쓰고
> 꼭지 끝에만 짧은 **에디터 한 줄**(펀치라인)을 둔다.

### Daytona Park / FREAK MAG — 디자인의 결
[daytona-park.com/freakmag](https://www.daytona-park.com/freakmag/)에서 가져온 것:
- **흑백 캔버스 + 포인트 컬러 하나**를 아껴 쓰는 절제
- **손그림(대충 그린 듯한)** 일러스트 → "한 사람이 직접 그렸다"는 체온
- **키치한 요소**(메탈 그라데이션 상단, 망가뜨린 듯한 로고)로 "농담할 줄 아는 사이트"라는 신호

## 디자인 시스템

### 핵심 원칙
> **99%는 절제된 흑백 미니멀, 1%는 의도적으로 망가뜨려 인간미를 준다.**
> 그 1%(손글씨 제호, 그라데이션, 손그림)는 *적게 쓸 때만* 효과가 있다. 도배하면 유치해진다.

### 컬러
- 바탕: **흑백** (검정 `#0d0d0d` / 흰색 `#ffffff` / 옅은 회색 `#f4f4f4`)
- 포인트: **노랑 `#ffd400`** — 한 화면에 두세 번만 등장시킨다.
  - *의미:* 오늘(아침 해), 떠오르는 산업, 개구쟁이 같은 느낌. 색을 감각이 아니라 의미로 골랐다.
  - 시장 연상색(비트코인 주황·이더리움 보라·상승 초록·하락 빨강)을 피해 중립적 정체성을 지킨다.

### 타이포그래피
- 제호: **손글씨**(Gaegu) — "오늘의 크립토". 나중에 *내 실제 손글씨*를 디지털화하면 유일무이해진다.
- 본문: **Gothic A1** (산세리프)
- 보조·라벨·인용: **Space Mono** (모노스페이스, 날것 HTML 느낌)
- 에디터 한 줄: **얇은 손글씨**(Gaegu 300) + 노란 'Editor' 라벨

### 시그니처 요소
- **메탈 그라데이션 상단 띠** + 흐르는 티커 — 순백 배경의 심심함을 막는다.
- **손그림 SVG 일러스트** — 꼭지마다 한 점. 디파이 해부엔 손그림 화살표 다이어그램(= 시그니처).
- **에디터 한 줄** — 꼭지 끝, 노란 'Editor' 라벨 + 얇은 손글씨 펀치라인.

## 이중 언어: 한국어 & English

발행 시점부터 **한국어판·영어판을 함께** 내고, 페이지 내 **KO / EN 토글**로 전환한다.

### 번역 철학 — "직역"이 아니라 "형제"
말장난·뉘앙스는 직역에서 죽는다. 두 판본을 **"같은 뉴스를 각자 언어로 다시 쓴 형제"**로 다룬다.

### 영어판 = 영작 학습의 장
자동 번역으로 뽑지 않는다. **발행 자체를 영어 수업으로** 만든다.
- 한국어 원고를 먼저 다듬는다.
- 영어판은 **편집자가 먼저 초벌 영작** → **첨삭 + 왜 그렇게 고쳤는지 설명**.
- 작업용 화면에만 "내 초안 / 첨삭본" 토글을 두어 실력 변화를 로그로 남긴다 (발행본엔 숨김).
- 배경: 토종 한국인. 크립토 업계에서 일하려면 영어 회화가 필수라 보고 지금부터 연습.
- 영어 낭독 **팟캐스트**도 함께 운영 계획. → 직접 쓴 문장이라야 입에 붙는다.

## 편집 방향 (잠정 — 발행하며 찾아가는 중)

목소리는 책상이 아니라 수십 호를 쓴 뒤 **발견하는 것.** 방향은 열어두되 아래를 의식적으로 시도한다.

1. **인문학적 시선** — "무슨 일이 있었나"가 아니라 "이 일이 무슨 의미인가".
2. **디파이 메커니즘 해부** — 복잡한 구조를 한 단계씩, 손그림과 함께. *단순 요약과 갈리는 핵심.*
3. **인물 발언 큐레이션** — 발언 요약 + "왜 지금 이 말을?" (1번 시선을 얹음).

> 차별점이 약한 3번은 1·2번에 묻어가는 재료로 쓰고, **1·2번을 더 자주 시도**한다.

## 페이지 구조

```
[메탈 그라데이션 띠 + 흐르는 티커]
마스트헤드 (손글씨 제호 + KO/EN 토글)
발행 정보 줄 (호수 배지 + 날짜)
─────────────────────────
꼭지 01  메인 뉴스
  번호 → 제목 → 손그림 → 본문(한 목소리로) → 인용 → +곁가지 → [Editor 한 줄]
꼭지 02  디파이 메커니즘 해부 (손그림 다이어그램)
꼭지 03  인물 발언 + 인문학적 시선
꼭지 04  짧은 소식 묶음
─────────────────────────
푸터 (구독 CTA + 콜로폰)
```
> 꼭지 배치는 고정이 아니라 발행하며 유동적으로 조정한다.

## 발행 · 배포

- **본진:** 직접 구매한 도메인에 직접 호스팅 (정적 HTML).
- **유입 채널:** 카카오톡 / 텔레그램 / 디스코드 / 트위터(X). 사이트가 본진, 채널은 "새 호 + 링크" 통로.
- **이메일 뉴스레터는 보류** — 초기엔 채널 유입이 더 현실적.
- 버전 관리: **Git + GitHub** (별도 git 노트 참고).
- 배포 다음 단계: GitHub → Cloudflare Pages / Vercel 연결 (push 시 자동 갱신).

---

## 작업 방식 (호 발행 루틴)

1. **소스 지도**로 재료를 줍는다 (3층 구조: 1차 소스 → 산업 내부 관점 → 속보 레이더).
   - 속보에서 발견하면 **1차 소스 원본을 확인**한 뒤 내 시선으로 가공한다.
2. 함께 검색·대화하며 그날 꼭지를 추린다.
3. **한국어 원고**를 먼저 다듬는다 (한 목소리 흐름 + 꼭지 끝 Editor 한 줄).
4. **영어판**을 직접 영작 → 첨삭받으며 완성.
5. 완성된 호를 페이지에 반영해 발행.

> 별도 파일: `source-map.md`(재료 창고), git 노트(버전 관리), 호별 초고.
